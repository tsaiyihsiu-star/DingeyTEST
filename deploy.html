從零開始部署待辦事項 App

這份指南會帶你一步步完成：建立 Google 試算表作為資料庫、部署 Apps Script 作為後端 API、把網頁傳到 GitHub Pages 對外發佈、最後在手機上「加到主畫面」變成 PWA。整個流程不需要寫程式、不需要付費。

⏱️ 約 30–45 分鐘 💰 完全免費 📱 支援 iOS / Android / 桌面 🔌 離線可用

1前置準備

動工前先確認你有以下三樣東西。如果有缺，先去申請：

  • Google 帳號：拿來建立試算表跟 Apps Script。建議用個人 Gmail，公司帳號有時會被 IT 政策擋。
  • GitHub 帳號：免費註冊。前往 github.com 用 email 註冊就好。
  • 四個檔案：todo.html（主程式）、manifest.json（PWA 設定）、sw.js（離線快取）、icon.png（App 圖示，512×512 PNG）。另外還會用到一個 Code.gs（GAS 後端程式碼）。
💡 沒有圖示怎麼辦？
暫時可以用任何 512×512 的 PNG 替代，例如從 Canva 做一個，或先用一張純色圖。圖示影響的只是桌面外觀，不影響功能。

2架構說明

動手之前，先理解這四個元件怎麼配合。理解架構後，後面每一步在做什麼就會很清楚。

元件角色放在哪
Google 試算表資料庫，儲存所有任務與分類Google Drive
Apps Script (Code.gs)後端 API，前端透過它讀寫試算表綁在試算表上
todo.html前端介面（你看到的整個 App）GitHub Pages
sw.js + manifest.json讓網頁變成 PWA，可離線、可加到主畫面GitHub Pages

資料流向

使用者操作流程
[手機 / 電腦瀏覽器] ↓ 開啟網址 [GitHub Pages] ← 提供 todo.html / sw.js / manifest.json ↓ 載入後執行 [todo.html (前端)] ← 使用者新增/編輯任務 ↓ HTTPS 請求 [GAS Web App URL] ← Code.gs 處理請求 ↓ 讀寫 [Google 試算表] ← 資料實際存放處
為什麼這樣設計？
1. 免費：三個服務（Google Sheets / Apps Script / GitHub Pages）全部免費。
2. 離線可用：sw.js 把網頁快取在裝置上，沒網路時也能看跟編輯，連線後自動同步。
3. 不需自己架伺服器：Apps Script 就是你的後端。

3建立 Google 試算表

這是你的資料庫。需要兩個工作表（sheet），欄位順序很重要，不能亂改。

1

到 sheets.google.com，點左上角 ＋ 空白 建立新試算表。

2

左上幫試算表命名，例如「待辦事項網頁資料庫」。名字隨便取，不影響功能。

3

把預設工作表名稱改成 Tasks（雙擊底部分頁名稱來改）。

4

在 Tasks 的第一列（A1 開始），從左到右依序填入以下標題：

欄位標題（一字不差）用途
ATask任務名稱
BCategory所屬分類
CDueDate截止日期
DStatusPending / Done
EPriority優先級數字
FNote備註
GmTime修改時間戳記
HID任務唯一識別碼
5

點底部 ＋ 新增第二個工作表，命名為 Categories。

6

在 Categories 的第一列填入以下標題：

欄位標題用途
ACategory Name分類名稱
BSortOrder排序值
CColor分類顏色（#RRGGBB）
DRowID分類唯一識別碼
EUpdatedAt更新時間戳記
⚠️ 標題不能改、欄位順序不能換
後端 Code.gs 是用「欄位位置」（A 欄、B 欄…）來讀寫資料的。標題拼錯或順序錯了會整個壞掉。如果不小心打錯，直接改回正確的就好。
7

（選用）可以先在 Categories 預填幾個分類，例如：

A: Category NameB: SortOrderC: ColorD: RowIDE: UpdatedAt
工作10#007AFF1（留空）
個人20#FF95002（留空）
家庭30#34C7593（留空）

不預填也可以，之後在 App 裡用「分類管理」新增。

8

記下試算表的網址，等下會用到。也可以先不記，反正等下會在 Apps Script 裡自動關聯。

4設定 Apps Script（後端）

Apps Script 是 Google 提供的免費伺服器環境，等於是你的後端 API。

1

在你剛建立的試算表上方選單，點 「擴充功能」→「Apps Script」。

2

會跳到新分頁，看到一個叫做 Code.gs 的檔案，裡面有幾行預設程式碼。把預設內容全部刪光。

3

把附帶的 Code.gs 內容整段貼進去。內容應該以這幾行開頭：

const SS = SpreadsheetApp.getActiveSpreadsheet();
const T_SHEET = SS.getSheetByName('Tasks');
const C_SHEET = SS.getSheetByName('Categories');

確認貼進去後，按 Ctrl S（Mac 為 ⌘ S）儲存。專案名稱可以隨便取，例如「待辦事項後端」。

4

第一次執行授權：在編輯器頂部找到函式下拉選單，選擇 getTasks，按 ▶ 執行。會跳出「需要授權」的提示。

  1. 點「審查權限」
  2. 選擇你的 Google 帳號
  3. 如果看到「Google 尚未驗證這個應用程式」的警告，點下方小字「進階」→「前往「未命名專案」（不安全）」
  4. 最後點「允許」
💡「不安全」是什麼？
因為這是你自己寫的腳本，沒經過 Google 審核，所以會顯示這個警告。實際上是安全的（程式碼是你自己貼的）。
5

授權完成後，下方「執行記錄」應該顯示「執行完畢」。如果出錯，看下面排錯區塊。

6

（強烈建議）設定每日備份：Code.gs 已內建 dailyBackup() 函式。設定方法：

  1. 在 Apps Script 編輯器左側點 ⏰ 觸發條件（鬧鐘圖示）
  2. 右下角點 ＋ 新增觸發條件
  3. 函式選 dailyBackup、事件來源選「時間驅動」、類型「日計時器」、時間選你想要的時段
  4. 儲存。每天會自動備份試算表並保留 7 天。

5部署 Web App 取得 URL

這一步是把你的 Apps Script 變成可以被 HTML 呼叫的 API。

1

在 Apps Script 編輯器右上角，點 「部署」→「新增部署作業」。

2

在跳出視窗左邊，點齒輪圖示 ⚙️ 選取類型 → 網頁應用程式。

3

右邊設定：

欄位值
說明隨便填，例如「待辦事項 v1」
執行身分我（你的 Google 帳號）
具存取權的使用者所有人
⚠️「所有人」會不會洩漏資料？
「所有人」指的是任何知道網址的人都能呼叫這個 API。但網址本身是一串很長的亂碼，外人猜不到。再者，這個 API 只能存取你綁的那份試算表，不會洩漏其他資料。如果擔心，可以在 Code.gs 加上自定義的 token 驗證（進階功能，本教學不涵蓋）。
4

點 「部署」。如果跳出授權視窗，再授權一次（同樣流程）。

5

部署成功後會顯示一段 「網頁應用程式」URL，長得像這樣：

https://script.google.com/macros/s/AKfycbz...很長的亂碼.../exec

把這段 URL 完整複製下來，貼到記事本暫存。等下要貼進 todo.html。

💡 測試 URL 有沒有通
把這段 URL 貼到瀏覽器網址列開啟，後面加上 ?action=getTasks，應該會回傳 [] 或一個任務陣列。如果回傳 HTML 錯誤訊息，代表部署有問題（通常是授權沒過）。

6建立 GitHub Repo 上傳檔案

GitHub 提供免費的網頁託管（GitHub Pages），不用自己架伺服器。

1

到 github.com 登入，點右上角 ＋ → New repository。

2

填寫設定：

欄位建議值
Repository nametodo 或 my-todo（英數小寫）
Description選填
Public / PrivatePublic（GitHub Pages 免費版需要 Public）
Initialize with README✓ 勾選

按下 Create repository。

3

進入 repo 頁面後，點 「Add file → Upload files」。

4

把以下四個檔案一次拖進去：

  • index.html（把 todo.html 改名成 index.html 再上傳，這是 GitHub Pages 的預設首頁）
  • manifest.json
  • sw.js
  • icon.png（512×512 PNG）

下方寫個 commit message（例如「初次上傳」），按綠色 Commit changes。

⚠️ 注意：檔名一定要叫 index.html。如果叫 todo.html，網址會變成 ...github.io/todo/todo.html，加到主畫面時 PWA 的 start_url 設定會錯亂。

7開啟 GitHub Pages

1

在 repo 頁面點上方 Settings。

2

左側選單往下找 Pages。

3

Source 選 Deploy from a branch。
Branch 選 main、資料夾選 / (root)，按 Save。

4

等待 1–2 分鐘後重新整理，頁面上方會顯示：

Your site is live at https://你的帳號.github.io/repo名稱/

記下這段網址，這就是你的待辦事項 App 對外網址。

💡 第一次部署要等多久？
通常 1–2 分鐘，最長可能 10 分鐘。如果一直顯示「正在建置」，可以去 repo 上方的 Actions 分頁看建置狀態。

8把 GAS URL 貼回 todo.html

這一步把前端跟後端「接起來」。

1

在 GitHub repo 裡點開 index.html，再點右上角鉛筆圖示 編輯。

2

用瀏覽器搜尋（Ctrl/⌘ + F）找這一行：

const GAS_URL = "https://script.google.com/macros/s/...../exec";
3

把雙引號裡的舊網址整段替換成你在 步驟 5 取得的 GAS Web App URL。

4

右上角 Commit changes 儲存。等 1 分鐘後，到你的 GitHub Pages 網址打開，App 應該就能正常運作了。

⚠️ 改完還是讀不到資料？
清一下瀏覽器快取（或無痕視窗開啟）。Service Worker 會把舊版檔案快取住，需要強制更新。詳見「後續更新」段落。

9在手機上加到主畫面

到這裡你的 App 已經上線了，但用瀏覽器開啟還是會看到網址列。讓它真正像原生 App 的最後一步：加到主畫面。

iOS（iPhone / iPad）

1

用 Safari 開啟你的 GitHub Pages 網址（Chrome 不支援這功能）。

2

點底部正中央的 分享按鈕（方框朝上箭頭）。

3

選單往下滑找到 「加入主畫面」 → 右上角 「加入」。

Android

1

用 Chrome 開啟網址。

2

右上角選單（三個點）→「安裝應用程式」或「新增至主螢幕」。

💡 加到主畫面後有什麼不同？
從桌面圖示開啟，會進入全螢幕模式（沒網址列），Service Worker 全力運作，等於是離線可用的原生 App 體驗。

10後續更新與版本管理

之後你想改 todo.html 或 Code.gs 時的標準流程。

改 todo.html

  1. 在 GitHub repo 編輯 index.html，儲存 commit。
  2. 同步修改版本號：把 index.html 裡的 APP_VERSION 跟 sw.js 裡的 CACHE_NAME 改成相同的新版本（例如 todo-v396）。
  3. GitHub Pages 1–2 分鐘後會更新。
  4. 使用者下次打開 App，Service Worker 偵測到新版本後會自動更新快取。
⚠️ 不改版本號 = 使用者拿到的還是舊版！
Service Worker 用 CACHE_NAME 判斷是否要更新快取。版本號沒變的話，它會以為內容沒變，永遠用舊的。每次發版都要改。

改 Code.gs

每次改完 Code.gs 都要重新部署，不然修改不會生效：

  1. Apps Script 編輯器右上角 「部署」→「管理部署作業」。
  2. 點現有部署右上角的 ✏️ 鉛筆圖示。
  3. 版本下拉選 「新版本」，按 「部署」。
💡 為什麼要選「新版本」？
如果建立新部署作業，URL 會變，前端 GAS_URL 就要重新貼。選「新版本」可保留同一個 URL，省事。

清除使用者裝置上的舊快取

如果有人說「我看到舊版」，請對方：

  • iOS：把 App 從主畫面刪除，重新加到主畫面。
  • 桌面：開發者工具 → Application → Service Workers → Unregister，然後重整。
  • 或在 App 內的「同步」面板按「重新載入」（已經是強制 reload）。

11常見問題排查

網頁打得開但任務都讀不到

檢查順序：

  1. 在 App 內展開側欄，點「🔄 同步」，看「同步記錄」有沒有錯誤訊息。
  2. 把 GAS_URL 後面加 ?action=getTasks 直接用瀏覽器開，看是否回傳資料。
  3. 如果回傳 HTML 錯誤頁，代表 Apps Script 沒授權或部署沒設「所有人」。
  4. 檢查試算表標題列是否完全一致（Task、Category… 不能拼錯）。
新增任務沒同步到雲端

同步記錄裡會看到「佇列 X 筆未同步」。常見原因：

  • 離線：恢復網路後會自動推送。
  • GAS URL 錯：檢查是否拼錯或忘記更新。
  • GAS 部署需要重新授權：到 Apps Script 重新部署一次。
加到主畫面後變空白頁

幾乎都是 manifest.json 的 start_url 路徑錯了。

如果你的網址是 https://abc.github.io/my-todo/，那 start_url 應該是 ./index.html（用相對路徑就對了）。預設值 ./index.html 通常沒問題。

iOS 加到主畫面後沒有圖示

檢查 icon.png：

  • 必須是 PNG 格式（不能是 JPG 或 SVG）
  • 建議尺寸 512×512
  • 檔案要放在 repo 根目錄
  • 確認 index.html 的 <link rel="apple-touch-icon" href="icon.png"> 路徑正確
分類顏色設定後沒同步到雲端

檢查 Categories 工作表的 C 欄標題是否正確命名為 Color。如果之前少建這欄，把 C 欄補上即可。

Excel 匯入完成後手機看不到

匯入是寫進佇列等待同步。等待約 5 秒讓推送完成，再到手機端點「同步」拉取最新資料。

想要密碼保護不開放給陌生人

三種做法：

  1. 讓 repo 變 private：但 GitHub Pages 免費版必須 Public，所以要付費 GitHub Pro。
  2. 在 Apps Script 加 token 驗證：每次請求帶一個 secret token，後端比對。需要寫 code。
  3. 前端密碼門：在 index.html 加密碼登入畫面（這個版本剛拿掉了，可以加回來）。能擋住誤入者，但不防駭客。
Apps Script 報 "Exceeded maximum execution time"

GAS 單次執行上限 6 分鐘。通常是因為試算表資料過多。建議：

  • 定期清理已完成超過 6 個月的任務（App 內已內建 autoCleanDone 自動清理）
  • 確認 dailyBackup 沒被重複觸發
我想加自己的 logo / 改顏色 / 改 App 名稱

分別改這些地方：

  • App 名稱：manifest.json 裡的 short_name 跟 name；index.html 裡的 <title> 跟 apple-mobile-web-app-title。
  • 主題色：index.html 開頭 :root 區塊裡的 --primary 等變數。
  • 圖示：替換 icon.png（記得改版本號讓快取更新）。
完成全部步驟後，你就有一個免費、可離線、隨時可改的 PWA 待辦事項 App 了。
